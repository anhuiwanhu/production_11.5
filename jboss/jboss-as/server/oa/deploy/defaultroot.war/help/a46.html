4.6.1新增流程

点击“流程设置”，在右侧显示所有已经增加的流程。

点击“新增”按钮，弹出wed工作流设计器页面，如下图。

点击“流程设置”，弹出新增流程的页面，如下图。

l 所属分类：可以自己新建一个分类，将同一类型的流程放在分类下，点击所属分类，支持多选，保存后流程显示在多个分类下

l 流程ID：ID不能重复

l 流程名称：可以按照自己的意愿去给流程进行命名

l 适用范围：部署流程后，在范围内的用户才可以使用这个流程，不在范围内的用户看不见这个流程。

l 表单类型：选择“自定义表单”，则下方显示“使用表单”，在其中可以选择在自定义表单中设置的流程表单；如果选择“JSP文件”，则下方显示“流程启动JSP文件”和“流程流转JSP文件”的预留功能，可上传开发好的JSP文件。

l 可写字段：发起流程时，没有设置设置为“可写字段”的字段内容，不可以填写或者修改，设置了“可写字段”，可以填写或修改字段内容

l 对发起人隐藏字段：发起流程与流程审批时，流程发起人在文件办理中打开流程页面，隐藏的字段内容屏蔽不显示。

l 流程办结抄送：勾选“需要”，页面展开办结抄送人的选择方式，选择参与人方式后，当流程办结完毕时，发送阅件给选择的参与人。

l 提醒项：指的是在文件办理中的“待办文件”等栏目的文件列表中，将这些提醒字段作为标题的一部分，以方便查询搜索。比如，提醒项选择“起始时间”，那么在待办文件中可能提醒“李心白的2011-11-11请假申请流程等待您的办理！”其中，“2011-11-11”就是这个流程中的“起始时间”，这样，我们按请假的起始时间查询就很方便。

l 提醒项预览：可以事先查看选择的提醒项在文件列表中的展现形式，以方便用户按照自己的使用习惯进行调整。

l 办理期限：用于效率分析-超期流程排名（ezFLOW）中，列表中显示所有超过设定的办理期限的流程记录。

l 字段联动：在自定义表单中设置字段联动和属性关联，点击下拉框显示设置的字段联动和字段触发

l 办理查阅查看人：在范围内的用户可以看到办理查阅菜单并且可以看到设置的流程名称

l 办理查阅维护人：在范围内的用户可以对流程进行：打印、催办、邮件转发、删除、批量删除、导出、选种导出、强制结束操作。详见下文“办理查阅”说明。

l 流程管理员：在范围内的用户可以看到该流程的所有流程记录

l 设置项-归档：如果选择，那么本流程下所有文件流转完毕，会自动归档到ezOFFICE系统的档案管理模块中。

l 设置项-流程办结后发起人可以打印：如果选择，那么发起人在我的办结文中打开此文件后显示打印按钮，可以进行打印操作。

l 设置项-流程办理结束时邮件提醒：如果选择，那么在流程办理结束后自动发送邮件给流程启动人。

l 设置项-在表单上保留退回意见：如果选择，那么在退回时表单中的批示意见字段内容被保留。

l 设置项-批示意见不能为空：如果选择，那么在流程办理过程中所有的批示意见必须填写，若不填写，系统提示请输入批示意见。

l 设置项-批示意见允许上传附件：如果选择，那么在流程办理过程中批示意见下方显示上传附件按钮，可以上传附件。

l 设置项-批示意见排序：

u 按时间顺序显示：如果选择，那么批示意见内容按照时间先后顺序排列

u 按时间倒序显示：如果选择，那么批示意见内容按照时间先后倒序排列

u 按职务顺序显示：如果选择，那么批示意见内容按照职务高低顺序排列

u 按职务倒序显示：如果选择，那么批示意见内容按照职务高低倒序排列

l 流程办结后可以打印 ：如果选择，那么流程办结后，流程办理人和参与者在文件办理中都可以打印流程。

l 活动参与者为空自动跳转下一步：如果选择，那么流程在审批过程中，发送至下一活动环节时，若下一活动环节无匹配的办理人，则直接跳过此活动环节至后续的活动环节。

l 同步到手机端：如果选择，那么在手机上可以审批此流程。

l 同步到PAD端：如果选择，那么在PAD中可以发起和审批此流程。

l 流程说明：对于流程应用的一个说明，输入内容后可以点击流程前面的图标进行查看，如下图

4.6.2设置活动

流程新增完毕，在web工作流设计器中增加流程的活动，比如上级经理审批、行政人事部审核等。

注意：发起人提交表单的操作，不算一个活动。比如我要请假，提交一个请假流程，然后提交给上级领导审批。这样，第一个活动是“上级领导审批”，作为发起时填写请假单的操作不需要作为一个活动来设置。

1. 选择活动类型：点击用户任务、自动返回任务等活动，然后点击页面，那么页面中就会多出一个相应的活动图标

2. 活动设置：双击新增的活动，就会弹出活动设置页面

3. 连接活动：点击连接，按照自己的思路将活动图标进行连接，那么在流程流转过程中就会按照图中显示的连接方法进行流转

注意：设置后需要进行保存，若设置后没有点击保存就关闭当前页面或点击关闭图标按钮，那么系统会弹出提示

活动类型

l 用户任务：相同于老工作流中的标准活动。

l 自动返回任务：只能连接在互斥网关后面

l 并行网关：并行网关后可以同时连接几个流程活动，所连接的活动必须全部都要办理完毕才能流转到下一步

l 互斥网关：互斥网关后可以同时连接几个流程活动，通过设置的虚拟条件，对流转方向进行多条件的自动判断，当同时又多个活动满足条件时，全部显示但是审核只能选择其中一个活动进行办理。

比如，1万<借款金额<5万的，到分管副总审批，这样就需要使用虚拟活动了。另外，在ezOFFICE系统中，发起流程的时候是不能在多个后继活动中选择某个活动的，而实际业务又需要在多个活动之间选择后继活动的，可以使用虚拟活动。比如一开始就判断发起人的职务是经理还是普通员工，如果是普通员工，发起时提交给部门领导，再到分管领导；如果是部门经理，就直接提交给分管领导。这样可以通过虚拟活动，在发起时就根据职务判断，自动流转到对应的后继活动。

l 包含网关：包含网关后可以同时连接几个流程活动，通过设置的虚拟条件，对流转方向进行多条件的自动判断，当同时又多个活动满足条件时，全部显示，审核能选择多个活动进行办理。

l 调用过程：(子过程活动)嵌入在流程中的子流程的活动。比如项目申报流程，先由分公司内部流程审核完毕，再提交到总公司的流程进行审核；在总公司流转的时候，不希望看到之前分公司的内部流转过程；总公司的这个流程又是可以独立流转的。符合这几个特征的情况下，在设置分公司流程的时候，可以将总公司的流程作为“子过程”。

l 子流程：嵌入在流程中的子流程，子流程来源于流程设置-子流程（ezFLOW）中。

9) 参与者：从操作方便的角度考虑，ezOFFICE系统设计了多种参与者的选择方式。流程的管理员可以根据实际业务情况来定义活动的参与者。

1.角色选择

l 从系统角色中指定：在下拉菜单中选择“AND”，则活动参与者范围是前后两者的交集，比如“财务部”（组织）的“部门经理”（角色）。如果不选择“AND”，即为空，则活动参与者范围是所选“角色”，如“部门经理”，如果选择组织级别，那么在前面设置的组织要满足这个级别的设置，否则在流程中就没有符合条件的办理人显示，如果选择了“AND”也选择了组织级别的话，那么就必须要满足全部的条件才能获取到活动办理人

在后面的下拉框中有“启动人相同办公地点”，这是一个很适用的选项。用户的办公地点在“人事管理→设置”中批量设置。

设置办公地点的一个重要作用之一，在于工作流程中准确定位活动参与者。比如A集团在全国有很多分公司，组织架构类似，现在有一个“分公司款项支付申请流程”适用于所有分公司。如果流程发起人是上海分公司的员工，那么在“分公司总经理审批”一步，他希望能直接显示自己分公司的总经理；如果流程发起人是浙江分公司的，那么在“分公司总经理审批”一步，他也希望能直接显示自己分公司的总经理。这样，流程管理员可以设置活动参与者为：角色（分公司总经理）AND启动人相同办公地点。

l 流程启动人：点击下拉框中，显示“本人”“部门领导”“分管领导”

u 选择“本人”后，在发送流程的时候会自动发送给自己

u 选择“上级领导”后：在发送流程时，流程会发送给流程启动人的上级领导，若流程启动人没有上级领导，那么流程发送时无法选择活动办理人，流程无法正常流转。

u 选择“部门领导”后：在发送流程时，流程会发送给流程启动人的部门领导，若流程启动人没有部门领导，那么流程发送时无法选择活动办理人，流程无法正常流转。

l 上一活动办理人的上级领导：应用情形举例——在流程中间环节，流转到部门员工小黄那里，小黄觉得这件事情需要请示他的上级领导，那么他的下一活动参与者可以设置为“上一活动办理人的上级领导”。

l 上一活动办理人的组织及下级组织：选择后在流程的流转过程中，活动办理人为上一活动办理人的组织及下级组织，如果选择了组织级别，那么只有满足组织级别设置的用户才能出现。

l 上一活动所有参与者：应用情形举例——上一步活动办理人是5个候选人中的小黄办理的，而下一活动也需要他办理，那么当前活动参与者可以设置为“上一活动所有参与者”。

l 选中组织：点击下拉框显示“分管领导”“部门领导”

u 选择“分管领导”：在流程流转过程中，选择下一活动办理人时，可以看到所有权限范围内能看到的组织，选中组织后，点击发送后，流程自动发送给选中组织的分管领导。

u 选择“部门领导”：在流程流转过程中，选择下一活动办理人时，可以看到所有权限范围内能看到的组织，选中组织后，点击发送后，流程自动发送给选中组织的部门领导。

l 流程启动人上级组织AND职务级别：选中后，在后面的下拉框中可以设置条件，如图所示，只有符合所有设置的条件办理人才会出现在下一活动办理人的选择框中

2.用户组织群组

l 从所有用户中选择：在整个组织架构、所有用户中选择办理人。

l 从选定的群组中选择： 点击后面的按钮，可以选择群组，群组设置在系统设置中进行操作，若后面没有进行任何设置那么在选择下一活动办理人范围就在设置群组中选择，如设置AND条件那么要满足AND的所有条件才能选择活动办理人，例如：选择了群组A，并设置了AND.组织.启动人的组织.组织级别为1，那么只有满足所有的设置条件的用户才会显示出来以供选择。

l 从本级组织及其下级组织中选择：从上一活动办理人所在组织及其下级组织中选择办理人。

l 从候选人员中指定：从管理员设置的用户中选择办理人。

l 从选定的范围中选择：从管理员设置的组织和群组中选择办理人。

3.默认选择

l 指定全部办理人：将管理员设置的用户作为全部办理人，比如管理员设置了5个人，那么这5个人必须都办理，这个流程才可以流转下去。

l 由接口决定：用于和其他系统进行相接。

10) 批示意见：点击下拉框显示“无批示意见”“自动增加批示意见”“批示意见对应字段”

l 无批示意见：选择此选项后，在此活动时没有批示意见的输入框

l 自动增加批示意见：在自定义数据表的时候，并没有设置属性为“批示意见”的字段。那么活动设置时，可以选择“自动增加批示意见”，这样，当流程流转到这一步的时候，表单下方会自动显示表格，在这里可以输入办理意见。

l 批示意见对应字段：在自定义数据表时，如果将字段属性设置为“批示意见”，那么在这里会全部列出。选中某个字段名称，就表明当前活动办理人在这个字段里写批示意见。当流程流转到这一步的时候，这个字段自动为可写（可输入）状态，在这里输入批示意见提交流程之后，在表单上的这个单元格里，办理意见之后自动带上办理人的姓名和办理时间。

11) 需要阅件：如果选择“需要”，则在下方显示参与者的选择方式和“阅件意见”对应字段的设置。应用情形举例——假设当前活动为“主管领导审批”，主管领导在审批的同时可能会让其他分管领导阅一下。这时，管理员就可以设置需要阅件，阅件人的设置参照活动参与人说明。

12) 高级设置：选择“允许代办”，则如果当前活动办理人在“个人办公”→“个人设置”→“工作代理”中设置了工作代理，那么当授权人收到一个待办文件的时候，代理人也收到了一份待办文件。关于工作代理的详细说明，请参考《ezOFFICE用户使用手册》。

 

表单控制页签：

表单控制主要针对字段中读写控制、签章保护字段控制、隐藏字段设置，勾选需要设置的字段，点击保存，设置成功。

a) 写字段：在审批流程时，办理人不能修改设置为“读字段”的字段内容，可以修改设置为“写字段”的字段内容。

b) 签章保护字段：在某个字段签章之后，就是对自己填写意见的确认，如果其他人修改了我的意见，那么设置了签章保护的字段中的签章就失效了。

c) 隐藏字段设置：设置了隐藏字段之后，这个活动办理人就看不到该字段的内容了。应用场景：对某个活动办理人比较敏感或需保密的字段可以设置隐藏。比如合同净利润评估，在流转过程中只给财务部审批这一步看见净利润，而不给其他活动办理人查看。【注意】隐藏字段不含批示意见字段。

4.6.4复制流程

如要新建的流程与已有流程类似，只有些许的变更，那么我们就可以考虑使用复制流程这个功能，复制流程后稍做改动。勾选一个或多个需要复制的流程，点击列表上方“复制流程”按钮即可。

4.6.5导入、导出流程

可以实现不同OA系统间的流程数据共享，例如：管理员可以把在测试环境OA系统中已编辑好的流程导出后导入至正式生产环境中使用。又如某A单位的OA系统创建了一个出差报销流程，A单位管理员可以把流程导出，发送给B单位管理员，B单位管理员可以导入到自己的OA系统中使用。注意：B单位的出差报销流程导入到系统后，流程使用的表单和适用范围等数据都是A单位的，所以需要把流程的表单和适用范围等数据重新编辑后才可正常使用。下面介绍具体的操作步骤：

1. 导出流程

勾选要导出的流程记录，点击列表右上方的导出按钮，下载到本地，流程记录以压缩文件方式存储在本地。

2. 导入流程

解压缩导出的流程压缩包，选择流程分类后，点击导入按钮，选择要导入的流程数据，导入成功后，在列表中显示。

4.6.6子流程（ezFLOW）

针对大集团在流程创建过程中存在很多相同的审批环节，此时可以将公共环节部分作为子流程放到流程中，当公共审批环节发生变更时，只需要在子流程中修改一次，既可以同步更新至所有调用了此子流程的主流程中，从而节省了繁复的工作量。
