  • 设置活动>

4.4.1新增流程

点击“流程设置”，在右侧显示所有已经增加的流程。

点击“新增”按钮，弹出新增流程的页面，如下图：

l 适用范围：在范围内的用户才可以使用这个流程，不在范围内的用户看不见这个流程。

l 所属分类：可以自己新建一个分类，将同一类型的流程放在分类下

l 流程名称：可以按照自己的意愿去给流程进行命名

l 流程类型：“业务流程”指流程按设置的活动进行流转。“随机流程”不需要定义流程活动，具有很强的随意性，根据当前用户的主观判断去选择下一办理人（只能选择一个办理人），可以记录流转轨迹。

l 归档：如果选择，那么本流程下所有文件流转完毕，会自动归档到ezOFFICE系统的档案管理模块中。

l 流程办结后可以打印：如果选择，流程的发起人和参与者在各文件办理页面都可以使用“打印”按钮。

l 批示意见不能为空：如果选择，那么在流程办理过程中所有的批示意见必须填写，若不填写，系统提示请输入批示意见。

l 在表单上保留退回意见：如果选择，那么在退回时表单中的批示意见字段内容被保留。

l 表单类型：选择“自定义表单”，则下方显示“使用表单”，在其中可以选择在自定义表单中设置的流程表单；如果选择“JSP文件”，则下方显示“流程启动JSP文件”和“流程流转JSP文件”的预留功能，可上传开发好的JSP文件。

l 发起时不填的字段：指流程发起时，发起人对这些选中的字段不可输入。

l 对发起人隐藏字段：如果一些字段中设置了默认值，那么在流程发起时，流程启动人无法看到设置的默认值

l 提醒项：指的是在文件办理中的“待办文件”等栏目的文件列表中，将这些提醒字段作为标题的一部分，以方便查询搜索。比如，提醒项选择“起始时间”，那么在待办文件中可能提醒“李心白的2011-11-11请假申请流程等待您的办理！”其中，“2011-11-11”就是这个流程中的“起始时间”，这样，我们按请假的起始时间查询就很方便。

目前系统支持以下编辑方式设置提醒项：单行文本、多行文本、自动编号、单选、多选、单选弹出选择、多选弹出选择、单选人、多选人、单选组织、多选组织、下拉框、时间、日期、时间日期、金额、金额大写、计算字段、子表金额合计。

l 提醒项预览：可以事先查看选择的提醒项在文件列表中的展现形式，以方便用户按照自己的使用习惯进行调整。

l 启动字段联动设置：参照启动字段联动设置视频

l 启动属性关联设置：参照启动属性关联设置视频

l 办理查阅查看人：在范围内的用户可以看到办理查阅菜单并且可以看到设置的流程名称

l 办理查阅维护人：在范围内的用户可以对流程进行：打印、催办、邮件转发、删除、批量删除、导出、选种导出、强制结束操作。详见下文“办理查阅”说明。

l 流程管理员：在范围内的用户可以看到该流程的所有流程记录

l 描述：对于流程应用的一个说明，输入内容后可以点击流程前面的图标进行查看，如下图

 

    接口页签下菜单：

l 表单处理类：开发接口。

l 表单提交方法：开发接口。

l 流程结束方法：开发接口。

l 脚本文件：开发接口。

l 手机脚本文件：开发接口。

l 页面加载方法名：开发接口。

l 页面保存方法名：开发接口。

4.4.2设置活动

流程新增完毕，在这个流程的操作栏点击活动定义按钮，然后在右侧点击“新增”按钮，或点击进行

流程及活动设置，比如上级经理审批、行政人事部审核等。

注意：发起人提交表单的操作，不算一个活动。比如我要请假，提交一个请假流程，然后提交给上级领导审批。这样，第一个活动是“上级领导审批”，作为发起时填写请假单的操作不需要作为一个活动来设置。

定义活动的页面如下图。分为三个页签：基本信息、操作按钮、接口。

1、“基本信息”页签

在基本信息页签，相关内容说明如下：

（1） 活动类型：

l 标准活动：相对于其他活动类型而言的最常用的活动。

l 子过程活动：嵌入在流程中的子流程的活动。比如项目申报流程，先由分公司内部流程审核完毕，再提交到总公司的流程进行审核；在总公司流转的时候，不希望看到之前分公司的内部流转过程；总公司的这个流程又是可以独立流转的。符合这几个特征的情况下，在设置分公司流程的时候，可以将总公司的流程作为“子过程”。

l 虚拟活动：对流转方向进行多条件的自动判断。比如，1万<借款金额<5万的，到分管副总审批，这样就需要使用虚拟活动了。另外，在ezOFFICE系统中，发起流程的时候是不能在多个后继活动中选择某个活动的，而实际业务又需要在多个活动之间选择后继活动的，可以使用虚拟活动。比如一开始就判断发起人的职务是经理还是普通员工，如果是普通员工，发起时提交给部门领导，再到分管领导；如果是部门经理，就直接提交给分管领导。这样可以通过虚拟活动，在发起时就根据职务判断，自动流转到对应的后继活动。

l 自动返回活动：无后继和结束活动的活动，办理人提交后自动返回到上一活动办理人。

 

（2） 子过程、子过程类型：活动类型选择“子过程活动”后，下方显示“子过程”和“子过程类型”。

l 子过程：在这里选择已经建立的独立流程作为子过程。

l 继承主流程数据:当主流程和子流程中有相同的字段时，在发起子流程时，那么主流程字段中的数据自动显示在子流程的相同字段上。

l 子过程类型：“串行子过程”是指子过程流转完毕，主流程才可以继续流转。“并行子过程”是指主流程和子过程可以同时流转。

 

（3） 活动走向：

l 由当前用户决定活动走向：如果后继有多个活动供选择的时候，那么由当前用户主观决定选择哪个后继活动。比如，某个活动后继有两个选择“分管领导审批”和“总经理审批”，那么当前用户就根据实际情况，自己判断并选择其中一个后继。

l 由流程决定活动走向：根据条件判断自动决定后继活动。比如，某个活动后继有两个可能“分管领导审批”和“总经理审批”，那么活动自动判断借款金额小于1万元的，到“分管领导审批”，大于1万元的到“总经理审批”。具体设置见下文“活动关系设置”。

（4） 办理方式：单人、单个多人、多人串行、多人并行。其含义见《ezOFFICE用户使用手册》工作流程部分说明。

（5） 使用表单：“默认”指此活动使用的表单和此流程使用表单一样。“自定义”是指此活动使用单独定义的表单，和其他活动使用表单不一样；在此要注意的是，选择的自定义表单必须是和发起时的表单用的同一张数据表形成的，否则无法选择。例如：表单1和表单2都是关联数据表1来进行设计的，只是2个表单在流程中的表现形式不一样。

 

（6） 办理期限：

无：指的是此活动没有任何的期限约束。

固定：此活动的办理期限在设置的时间之内，比如上级领导审批的办理期限为1个小时，提前15分钟提醒办理人。

到期后处理：

即时通讯提醒：到达设置的办理期限及提前提醒时间时，给办理人发送即时消息（如：RTX、eLink等）；

短信提醒：到达设置的办理期限及提前提醒时间时，如果开通了短信提醒功能并办理人有电话号码，系统会自动发送短信提示；

催办：到达设置的办理期限及提前提醒时间时，系统会自动给办理人发送催办，办理人可在个人办公-催办管理-接收的催办中进行查看；

自动跳转：当超过设置的办理期限后，系统会自动跳转至下一活动办理人（下一活动办理人必须在下图显示范围内才有效）。

     自动跳转的流程记录，可以在效率分析下“自动跳转”模块中查看。参与办理的用户也可以点击此流程记录，在批示意见和“流程记录”页签中都会标识出“自动跳转”信息。

（7） 参与者：从操作方便的角度考虑，ezOFFICE系统设计了多种参与者的选择方式。流程的管理员可以根据实际业务情况来定义活动的参与者。

1.角色选择

l 从系统角色中指定：在下拉菜单中选择“AND”，则活动参与者范围是前后两者的交集，比如“财务部”（组织）的“部门经理”（角色）。如果不选择“AND”，即为空，则活动参与者范围是所选“角色”，如“部门经理”。当选择组织为“启动人的组织及下级组织、上一活动办理人的组织及下级组织”时，会显示组织级别选项，如果选择组织级别，那么在前面设置的组织按设置的级别向下查找，否则在流程中就没有符合条件的办理人显示；如果选择了“AND”也选择了组织级别的话，那么就必须要满足全部的条件才能获取到活动办理人

 

在后面的下拉框中有“启动人相同办公地点”，这是一个很适用的选项。用户的办公地点在“人事管理→设置”中批量设置。

设置办公地点的一个重要作用之一，在于工作流程中准确定位活动参与者。比如A集团在全国有很多分公司，组织架构类似，现在有一个“分公司款项支付申请流程”适用于所有分公司。如果流程发起人是上海分公司的员工，那么在“分公司总经理审批”一步，他希望能直接显示自己分公司的总经理；如果流程发起人是浙江分公司的，那么在“分公司总经理审批”一步，他也希望能直接显示自己分公司的总经理。这样，流程管理员可以设置活动参与者为：角色（分公司总经理）AND启动人相同办公地点。

l 流程启动人：选择后，在发送流程的时候会自动发送给自己

l 流程启动人的上级领导：在发送流程时，流程会发送给流程启动人的上级领导，若流程启动人没有上级领导，那么流程发送时无法选择活动办理人，流程无法正常流转。

l 流程启动人的部门领导：在发送流程时，流程会发送给流程启动人的部门领导，若流程启动人没有部门领导，那么流程发送时无法选择活动办理人，流程无法正常流转。

l 流程启动人的分管领导：在发送流程时，流程会发送给流程启动人的分管领导，若流程启动人没有分管领导，那么流程发送时无法选择活动办理人，流程无法正常流转。

l 上一活动办理人的上级领导：应用情形举例——在流程中间环节，流转到部门员工小黄那里，小黄觉得这件事情需要请示他的上级领导，那么他的下一活动参与者可以设置为“上一活动办理人的上级领导”。

l 上一活动办理人的组织及下级组织：选择后在流程的流转过程中，活动办理人为上一活动办理人的组织及下级组织，如果选择了组织级别，那么会从上活动办理人的组织及下级组织并从设置的组织级别向下查找，显示符合的办理人。

l 上一活动所有参与者：应用情形举例——上一步活动办理人是5个候选人中的小黄办理的，而下一活动也需要他办理，那么当前活动参与者可以设置为“上一活动所有参与者”。

l 活动参与者本人：选择后，在发送时，选择的办理人为选择的活动的参与者本人，例如：有活动A和B，在B活动设置时，在“活动参与者本人”后的下拉框中选择A活动，那么在上一活动往B发送时，办理人是就A活动的参与者本人

l 活动参与者上级领导：选择后，在发送时，选择的办理人为选择的活动的参与者本人，例如：有活动A和B，在B活动设置时，在“活动参与者本人”后的下拉框中选择A活动，那么在上一活动往B发送时，办理人是就A活动参与者的上级领导

l 选中组织的分管领导：在流程流转过程中，选择下一活动办理人时，可以看到所有权限范围内能看到的组织，选中组织后，点击发送后，流程自动发送给选中组织的分管领导。

l 选中组织的部门领导：在流程流转过程中，选择下一活动办理人时，可以看到所有权限范围内能看到的组织，选中组织后，点击发送后，流程自动发送给选中组织的部门领导。

l 流程启动人上级组织AND职务级别：选中后，在后面的下拉框中可以设置条件，如图所示，只有符合所有设置的条件办理人才会出现在下一活动办理人的选择框中

2.用户组织群组

l 从所有用户中选择：在整个组织架构、所有用户中选择办理人。

l 从选定的群组中选择： 点击后面的按钮，可以选择群组，群组设置在系统设置中进行操作，若后面没有进行任何设置那么在选择下一活动办理人范围就在设置群组中选择，如设置AND条件那么要满足AND的所有条件才能选择活动办理人，例如：选择了群组A，并设置了AND.组织.启动人的组织及下级组织.组织级别为2，那么就从当前用户所在组织及下级组织的第二级组织结构向下查找，将符合条件

的人员显示出来。

l 从本级组织及其下级组织中选择：从上一活动办理人所在组织及其下级组织中选择办理人。

l 从候选人员中指定：从管理员设置的用户中选择办理人。

l 从选定的范围中选择：从管理员设置的组织和群组中选择办理人。

    

3.默认选择

l 由表单中的某个字段值决定：当流程表单中含有选择用户的字段时，在此处设置选择表单中的选人字段，在后面的下拉框中选择相应的选项：

本人：例如表单中选择用户字段的值为用户A，在此处选择本人，那么流程流转至此活动时，下一步的活动办理人为A

上级领导：例如表单中选择用户字段的值为用户A，在此处选择上级领导，那么流程流转至此活动时，下一步的活动办理人为A的上级领导

部门领导：例如表单中选择用户字段的值为用户A，在此处选择部门领导，那么流程流转至此活动时，下一步的活动办理人为A的部门领导

分管领导：例如表单中选择用户字段的值为用户A，在此处选择分管领导，那么流程流转至此活动时，下一步的活动办理人为A的分管领导

l 指定全部办理人：将管理员设置的用户作为全部办理人，比如管理员设置了5个人，那么这5个人必须都办理，这个流程才可以流转下去。

l 由接口决定：用于和其他系统进行相接。

 

（8） 批示意见

l 批示意见对应字段：在自定义数据表时，如果将字段属性设置为“批示意见”，那么在这里会全部列出。选中某个字段名称，就表明当前活动办理人在这个字段里写批示意见。当流程流转到这一步的时候，这个字段自动为可写（可输入）状态，在这里输入批示意见提交流程之后，在表单上的这个单元格里，办理意见之后自动带上办理人的姓名和办理时间。

l 自动增加批示意见：在自定义数据表的时候，并没有设置属性为“批示意见”的字段。那么活动设置时，可以选择“自动增加批示意见”，这样，当流程流转到这一步的时候，表单下方会自动显示表格，在这里可以输入办理意见。

l 无批示意见：选择此选项后，在此活动时没有批示意见的输入框。

 

（9） 需要阅件：如果选择“需要”，则在下方显示参与者的选择方式和“阅件意见”对应字段的设置。应用情形举例——假设当前活动为“主管领导审批”，主管领导在审批的同时可能会让其他分管领导阅一下。这时，管理员就可以设置需要阅件，阅件人的设置参照活动参与人说明。

（10） 读写控制：在审批流程时，办理人不能修改设置为“读字段”的字段内容，可以修改设置为“写字段”的字段内容。

（11） 签章保护字段：在某个字段签章之后，就是对自己填写意见的确认，如果其他人修改了我的意见，那么设置了签章保护的字段中的签章就失效了。

（12） 隐藏字段设置：设置了隐藏字段之后，这个活动办理人就看不到该字段的内容了。应用场景：对某个活动办理人比较敏感或需保密的字段可以设置隐藏。比如合同净利润评估，在流转过程中只给财务部审批这一步看见净利润，而不给其他活动办理人查看。【注意】隐藏字段不含批示意见字段。

（13） 审批意见查看范围：指当前活动的办理意见哪些用户可以查看。应用场景1：公司领导要求各部门经理会签的时候，不能相互查看到办理意见，这样做的目的是促进各中层领导独立思考和决策，不被其他平行部门领导的意见所影响。应用场景2：分公司提交一个车辆采购申请，总公司审批结果是不同意，最后流程流转到分公司，但是不方便让分公司看到总公司的批示意见。

（14） 高级设置：选择“允许代办”，当前活动办理人在“个人办公”→“个人设置”→“工作代理”中设置了工作代理（在提交到设置了“允许代办”的活动之前创建的工作代理均有效），那么当授权人收到一个待办文件的时候，代理人也收到了一份待办文件。关于工作代理的详细说明，请参考《ezOFFICE用户使用手册》。

（15） 短信提醒：选中短信提醒后，系统默认的为“默认提醒”，当发送至此活动时，办理人设置的电话号码则会收到系统发送的提醒。

1、“基本信息”页签

在基本信息页签，相关内容说明如下：

（1） 活动类型：

l 标准活动：相对于其他活动类型而言的最常用的活动。

l 子过程活动：嵌入在流程中的子流程的活动。比如项目申报流程，先由分公司内部流程审核完毕，再提交到总公司的流程进行审核；在总公司流转的时候，不希望看到之前分公司的内部流转过程；总公司的这个流程又是可以独立流转的。符合这几个特征的情况下，在设置分公司流程的时候，可以将总公司的流程作为“子过程”。

l 虚拟活动：对流转方向进行多条件的自动判断。比如，1万<借款金额<5万的，到分管副总审批，这样就需要使用虚拟活动了。另外，在ezOFFICE系统中，发起流程的时候是不能在多个后继活动中选择某个活动的，而实际业务又需要在多个活动之间选择后继活动的，可以使用虚拟活动。比如一开始就判断发起人的职务是经理还是普通员工，如果是普通员工，发起时提交给部门领导，再到分管领导；如果是部门经理，就直接提交给分管领导。这样可以通过虚拟活动，在发起时就根据职务判断，自动流转到对应的后继活动。

l 自动返回活动：无后继和结束活动的活动，办理人提交后自动返回到上一活动办理人。

 

（2） 子过程、子过程类型：活动类型选择“子过程活动”后，下方显示“子过程”和“子过程类型”。

l 子过程：在这里选择已经建立的独立流程作为子过程。

l 继承主流程数据:当主流程和子流程中有相同的字段时，在发起子流程时，那么主流程字段中的数据自动显示在子流程的相同字段上。

l 子过程类型：“串行子过程”是指子过程流转完毕，主流程才可以继续流转。“并行子过程”是指主流程和子过程可以同时流转。

 

（3） 活动走向：

l 由当前用户决定活动走向：如果后继有多个活动供选择的时候，那么由当前用户主观决定选择哪个后继活动。比如，某个活动后继有两个选择“分管领导审批”和“总经理审批”，那么当前用户就根据实际情况，自己判断并选择其中一个后继。

l 由流程决定活动走向：根据条件判断自动决定后继活动。比如，某个活动后继有两个可能“分管领导审批”和“总经理审批”，那么活动自动判断借款金额小于1万元的，到“分管领导审批”，大于1万元的到“总经理审批”。具体设置见下文“活动关系设置”。

（4） 办理方式：单人、单个多人、多人串行、多人并行。其含义见《ezOFFICE用户使用手册》工作流程部分说明。

（5） 使用表单：“默认”指此活动使用的表单和此流程使用表单一样。“自定义”是指此活动使用单独定义的表单，和其他活动使用表单不一样；在此要注意的是，选择的自定义表单必须是和发起时的表单用的同一张数据表形成的，否则无法选择。例如：表单1和表单2都是关联数据表1来进行设计的，只是2个表单在流程中的表现形式不一样。

 

（6） 办理期限：

无：指的是此活动没有任何的期限约束。

固定：此活动的办理期限在设置的时间之内，比如上级领导审批的办理期限为1个小时，提前15分钟提醒办理人。

到期后处理：1.系统中提醒是指系统会自动发送一份邮件提示2.短信提醒如果开通了短信提醒功能并办理人有电话号码，系统会自动发送短信提示3.催办，系统会自动发送催办，办理人可以在系统中进行查看4.自动跳转，当超过期限后，系统会自动跳转至下一活动办理人，但是下一活动办理人必须在下图显示范围内才有效

 

（7） 参与者：从操作方便的角度考虑，ezOFFICE系统设计了多种参与者的选择方式。流程的管理员可以根据实际业务情况来定义活动的参与者。

1.角色选择

l 从系统角色中指定：在下拉菜单中选择“AND”，则活动参与者范围是前后两者的交集，比如“财务部”（组织）的“部门经理”（角色）。如果不选择“AND”，即为空，则活动参与者范围是所选“角色”，如“部门经理”，如果选择组织级别，那么在前面设置的组织要满足这个级别的设置，否则在流程中就没有符合条件的办理人显示，如果选择了“AND”也选择了组织级别的话，那么就必须要满足全部的条件才能获取到活动办理人

 

在后面的下拉框中有“启动人相同办公地点”，这是一个很适用的选项。用户的办公地点在“人事管理→设置”中批量设置。

设置办公地点的一个重要作用之一，在于工作流程中准确定位活动参与者。比如A集团在全国有很多分公司，组织架构类似，现在有一个“分公司款项支付申请流程”适用于所有分公司。如果流程发起人是上海分公司的员工，那么在“分公司总经理审批”一步，他希望能直接显示自己分公司的总经理；如果流程发起人是浙江分公司的，那么在“分公司总经理审批”一步，他也希望能直接显示自己分公司的总经理。这样，流程管理员可以设置活动参与者为：角色（分公司总经理）AND启动人相同办公地点。

l 流程启动人：选择后，在发送流程的时候会自动发送给自己

l 流程启动人的上级领导：在发送流程时，流程会发送给流程启动人的上级领导，若流程启动人没有上级领导，那么流程发送时无法选择活动办理人，流程无法正常流转。

l 流程启动人的部门领导：在发送流程时，流程会发送给流程启动人的部门领导，若流程启动人没有部门领导，那么流程发送时无法选择活动办理人，流程无法正常流转。

l 流程启动人的分管领导：在发送流程时，流程会发送给流程启动人的分管领导，若流程启动人没有分管领导，那么流程发送时无法选择活动办理人，流程无法正常流转。

l 上一活动办理人的上级领导：应用情形举例——在流程中间环节，流转到部门员工小黄那里，小黄觉得这件事情需要请示他的上级领导，那么他的下一活动参与者可以设置为“上一活动办理人的上级领导”。

l 上一活动办理人的组织及下级组织：选择后在流程的流转过程中，活动办理人为上一活动办理人的组织及下级组织，如果选择了组织级别，那么只有满足组织级别设置的用户才能出现。

l 上一活动所有参与者：应用情形举例——上一步活动办理人是5个候选人中的小黄办理的，而下一活动也需要他办理，那么当前活动参与者可以设置为“上一活动所有参与者”。

l 活动参与者本人：选择后，在发送时，选择的办理人为选择的活动的参与者本人，例如：有活动A和B，在B活动设置时，在“活动参与者本人”后的下拉框中选择A活动，那么在上一活动往B发送时，办理人是就A活动的参与者本人

l 活动参与者上级领导：选择后，在发送时，选择的办理人为选择的活动的参与者本人，例如：有活动A和B，在B活动设置时，在“活动参与者本人”后的下拉框中选择A活动，那么在上一活动往B发送时，办理人是就A活动参与者的上级领导

l 选中组织的分管领导：在流程流转过程中，选择下一活动办理人时，可以看到所有权限范围内能看到的组织，选中组织后，点击发送后，流程自动发送给选中组织的分管领导。

l 选中组织的部门领导：在流程流转过程中，选择下一活动办理人时，可以看到所有权限范围内能看到的组织，选中组织后，点击发送后，流程自动发送给选中组织的部门领导。

l 流程启动人上级组织AND职务级别：选中后，在后面的下拉框中可以设置条件，如图所示，只有符合所有设置的条件办理人才会出现在下一活动办理人的选择框中

2.用户组织群组

l 从所有用户中选择：在整个组织架构、所有用户中选择办理人。

l 从选定的群组中选择： 点击后面的按钮，可以选择群组，群组设置在系统设置中进行操作，若后面没有进行任何设置那么在选择下一活动办理人范围就在设置群组中选择，如设置AND条件那么要满足AND的所有条件才能选择活动办理人，例如：选择了群组A，并设置了AND.组织.启动人的组织.组织级别为1，那么只有满足所有的设置条件的用户才会显示出来以供选择。

l 从本级组织及其下级组织中选择：从上一活动办理人所在组织及其下级组织中选择办理人。

l 从候选人员中指定：从管理员设置的用户中选择办理人。

l 从选定的范围中选择：从管理员设置的组织和群组中选择办理人。

本人：例如表单中选择用户字段的值为用户A，在此处选择本人，那么流程流转至此活动时，下一步的活动办理人为A

部门领导：例如表单中选择用户字段的值为用户A，在此处选择部门领导，那么流程流转至此活动时，下一步的活动办理人为A的部门领导

分管领导：例如表单中选择用户字段的值为用户A，在此处选择分管领导，那么流程流转至此活动时，下一步的活动办理人为A的分管领导

l 指定全部办理人：将管理员设置的用户作为全部办理人，比如管理员设置了5个人，那么这5个人必须都办理，这个流程才可以流转下去。

（8） 批示意见

l 批示意见对应字段：在自定义数据表时，如果将字段属性设置为“批示意见”，那么在这里会全部列出。选中某个字段名称，就表明当前活动办理人在这个字段里写批示意见。当流程流转到这一步的时候，这个字段自动为可写（可输入）状态，在这里输入批示意见提交流程之后，在表单上的这个单元格里，办理意见之后自动带上办理人的姓名和办理时间。

l 自动增加批示意见：在自定义数据表的时候，并没有设置属性为“批示意见”的字段。那么活动设置时，可以选择“自动增加批示意见”，这样，当流程流转到这一步的时候，表单下方会自动显示表格，在这里可以输入办理意见。

l 无批示意见：选择此选项后，在此活动时没有批示意见的输入框。

 

（9） 需要阅件：如果选择“需要”，则在下方显示参与者的选择方式和“阅件意见”对应字段的设置。应用情形举例——假设当前活动为“主管领导审批”，主管领导在审批的同时可能会让其他分管领导阅一下。这时，管理员就可以设置需要阅件，阅件人的设置参照活动参与人说明。

（10） 读写控制：在审批流程时，办理人不能修改设置为“读字段”的字段内容，可以修改设置为“写字段”的字段内容。

（11） 签章保护字段：在某个字段签章之后，就是对自己填写意见的确认，如果其他人修改了我的意见，那么设置了签章保护的字段中的签章就失效了。

（12） 隐藏字段设置：设置了隐藏字段之后，这个活动办理人就看不到该字段的内容了。应用场景：对某个活动办理人比较敏感或需保密的字段可以设置隐藏。比如合同净利润评估，在流转过程中只给财务部审批这一步看见净利润，而不给其他活动办理人查看。【注意】隐藏字段不含批示意见字段。

（14） 高级设置：选择“允许代办”，则如果当前活动办理人在“个人办公”→“个人设置”→“工作代理”中设置了工作代理，那么当授权人收到一个待办文件的时候，代理人也收到了一份待办文件。关于工作代理的详细说明，请参考《ezOFFICE用户使用手册》。

（15） 短信提醒：选中短信提醒后，系统默认的为“默认提醒”，当发送至此活动时，办理人设置的电话号码则会收到系统发送的提醒。

在设置活动的“接口”页签，是用于开发的接口。
