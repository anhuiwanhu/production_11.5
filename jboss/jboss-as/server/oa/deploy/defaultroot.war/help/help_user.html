基础设置

系统管理模块是具有系统管理员或者普通管理员权限的用户才可以看到的功能模块，用于对整个系统进行基础的管理操作。

系统刚安装好的时候，只有账号admin（系统管理员）、security（系统安全员），密码是111111，用来进行系统设置、安全设置、系统日志维护；用户使用系统时在任何页面均看不到、选不到admin、security用户。

  • 系统设置>
  • 界面设置>
  • 登录页设置>
  • 基础数据设置>
  • 弹出窗口设置>
  • 接口设置

1.1.1系统设置

系统管理员点击菜单“系统管理”，默认进入“基础设置”>“系统设置”页面：

 

 附件上传：设置附件两种上传方式，FTP和HTTP，默认使用http上传。如果设置为FTP方式，需要搭建FTP服务器，具体设置请参考《系统安装手册》

 ;附件大小：设置是否限制上传的附件大小，默认不限制附件大小

登陆验证码：设置为使用时，则登陆页用户输入账号和密码后，还需输入验证码才可进行登录，默认不适用。

账号大小写：设置为不区分，则系统对账号大小写不进行区分，默认区分。

 手写意见（第3方产品）：是否启用手写意见。如果启用，系统中所有审批意见的地方显示“手写签名”。点击后，在意见输入框中可以手写，点击右上角的“签章”，可以调用在发文管理中设置的软件印章。默认不使用

 电子签章（第3方产品）：是否启用电子签章。如果启用了，系统中审批意见的地方显示“电子签章”。点击后，在右上角显示“盖章”，可以调用KEY中的印章，点击右上角的“签字”，可以手写签字。默认不使用

短信：是否启用手机短信提醒。短信提醒需要系统已经集成第三方通讯提供商的短信功能之后，才可以正常使用。如果启用短信，在短信模块、流程活动设置页面进行相应的设置后，在发送审批流程时可以看到“短信提醒”选项。默认不使用

在线感知：如果系统集成了沟通平台，如rtx、elink、ucstar等，那么系统中一些出现用户姓名的地方，如查看信息页面的“阅读情况”、在线用户列表等页面，自动“感知”客户端是否在线，在姓名左侧出现头像，点击可显示一些即时通讯常用功能，如“发送即时消息”。如果不启用在线感知，看不见头像。通过在线感知功能，可以很便捷的进行一些即时沟通。

附件在线预览：使用后，系统中所有有附件的地方，均有预览按钮，点击可在线对附件进行预览，默认不使用。点击“预览”按钮弹出页面如下，直接显示可以预览的内容，支持office格式、txt文件、图片（jpg、gif、png、bmp格式）的直接预览。

如果附件字段为可写时，之前上传的Word、excel、ppt格式文件后显示“在线编辑”按钮，可以使用“在线编辑”功能。

工作流引擎：系统中，提供了两套工作流引擎，在这里您可以进行实际业务需要来进行选择开启，控制整个OA中引用到工作流引擎的所有模块（除“物品管理”模块，物品管理只能使用老工作流引擎）。

WORD编辑：如果选择使用，则在信息管理→发布信息时，会显示“WORD编辑”，“EXCEL编辑”和“PPT编辑”。您还可以对附件大小进行控制，默认15M

工作流程超期判断：选择自然天则按照自然天数进行判断工作流程是否过期，选择工作日，旁边则会出现设置按钮，点击后弹出工作日设置页面，如下图所示：

根据您设置的工作日和节假日判断工作流程是否过期。

例如：在工作流程的活动中设置了组长审核的活动时间为2天，工作流程超期判断选择工作日，在工作日设置页面中设置了2011-11-21至2011-11-25为工作日，2011-11-26至2011-11-27为节假日。工作流程于2011年11月24日进行报送至组长审核活动，审核人查看待办文件时显示的办理期限为2011年11月28日。

工作日志维护：“不允许”，即工作日志增加后就不能删除。“允许     天之内维护”是指在设置的天数内可以执行修改和删除操作，在这个时间以外的日志不能执行修改删除。这个设计的管理意义在于保证日志的客观性，是客观记录，不能随意修改。

邮件群发：如果控制，用户未被赋予邮件权限的情况下，在发邮件选择收件人时，选不到组织和群组，只能选择用户。

历史邮件：如果内部邮件数据量非常大，则可以启用历史邮件，并设置邮件保留周期，超出设置期限的邮件自动转入历史邮件，这样可以减轻系统压力，改善系统性能。历史邮件只能查看，不能进行回复、转发等操作。

历史日志：系统操作日志，根据启用后设置的时间自动转移日志记录到历史日志。

Html编辑器默认字体：设置了默认字体之后，在系统中任何页面的Html编辑器中录入文字的时候，都默认这个字体。用户也可以再次修改字体。

与ezSITE结合：和万户ezSITE网站管理系统相结合，可以将ezOFFICE系统中的相关信息同步到网站上，避免重复管理。

邮件提醒方式：选择外部邮件需配置：SMTP服务器地址、电子邮件地址、账号和密码。正确配置后，系统工作中发出的邮件都会发到已经配置的外部邮箱里，选择内部邮件无需配置，工作邮件直接发到系统内部邮箱里。

容量设置：“内部邮件容量”和“我的文档容量”设置的是所有用户内部邮件和我的文档容量的默认大小，管理员在“用户管理”中，可以对具体用户进行容量的再次设置，最终以在“用户管理”中设置的容量大小为准。如果设置某个用户的我的文档容量为0，则此用户在“个人办公”下不能看到“我的文档”。“邮件封数”指用户收件箱、发件箱等所有文件夹中的邮件封数之和的最大限制数，如果超出，系统会提示删除旧邮件。

全部设置完毕后，点击底部保存按钮，即可保存您对系统的基本设置。

1.1.3登录页设置

在这里，您可以对系统登录页进行设置。系统中已经提供了不同季节不同节日的登录页，点击操作栏中启用按钮，则可更换登录页。

点击修改按钮，您可以对登录页进行自定义设置，按照系统提示上传相应图片后，点击保存退出，则可完成个性化登录页的设置。

点击列表右上方新增按钮，可以新增登录页，同修改页面一样，按照提示上传相应图片后点击保存退出，即可完成登录页新增。

1.1.4基础数据设置

1.1.1.1 工作时间设置

作为系统管理员，您可以在这里根据具体业务需要对组织的工作时间进行设置。具体设置项包括上班、下班、迟到、早退等数据。

具体设置方法如下：

（1）点击新增，弹出新增工作时间页面

（2）编辑基本信息：填写名称，制定适用范围，则适用范围内的组织和用户采用该设置的时间进行考勤管理，设置具体各时间项。

（3）设置完毕后，点击保存退出。

 

1.1.1.2 工作日设置

 作为系统管理员，您可以在这里进行工作日和节假日的设置，设置哪些时间段为工作日，哪些时间段为节假日。如果不设置的话，则考勤管理会默认工作周设置的工作时间计算。

具体设置方法如下：

（1）点击新增按钮，弹出新增页面

（2）选择日期，并且设置为工作日或者节假日，如9月6日到9月8日为中秋节假期，则可选择该时间段为节假日：

（3）点击保存退出

1.1.1.3 工作周设置

作为系统管理员，您可以在这里对一周的工作时间进行设置，设置一周那几天为工作日，那几天为休息日。系统进行考勤计算时，会首先参考工作日中设置的时间，如果工作日中未设置则以工作周中设置为准。

 

1.1.1.4 职务设置

 作为管理员，您可以在这里进行职务级别的基础设置，设置完成后，在用户新增或者修改时可进行调用。后面工作流程流转进行判断时，也会用到职务级别。

具体设置方法如下：

（1）点击“新增”按钮，弹出新职位页面：

（2）填写职位名称，设置职务级别。职务级别字段类型为浮点型，可以设置为小数。职务级别0为最大级别，数字越大职级越小。然后选择适用范围，这里的适用范围，指的是职务级别在哪个范围的组织内生效，比如下图设置为培训组织，则新增用户为培训组织时，选择职务才能够选到您设置的“主管”这一职务。同样，在进行判断时，培训组织的主管对应的用户，他的职务级别就是您这里设置的3

(3)点击保存退出，即可完成职务的设置。根据具体业务情况，将组织内所有职务级别一一进行设置即可。

1.1.1.5 办公地点分类设置

作为系统管理员，您可以在这里进行办公地点的分类设置，方便分子公司共用一套系统时的管理：

具体设置方法如下：

（1）点击新增按钮，弹出新增分类页面

（2）填写分类名称，排序码后，点击保存退出。即可完成办公地点分类设置。

对于已经存在的分类，您可以点击操作栏中修改按钮和删除按钮进行修改删除操作。

 

1.1.1.6 办公地点设置

创建完办公地点分类后，您可以在此进行办公地点的进一步设置，完善各分子公司信息：

具体设置方法如下：

（1）点击新增按钮， 弹出新增办公地点页面

（2）选择所属分类，完善具体信息。注意的是，这里的适用范围指的是哪些组织在这个办公地点办公。

对于已经增加的办公地点，您可以通过点击操作栏中修改或删除按钮，进行修改和删除操作。

1.1.6接口设置

接口设置包括用户接口，组织接口和登陆接口。这里设置的接口信息供ezEDI、及时通讯及第三方调用，如果接口信息设置了启用，那么第三方用户就可以通过连接信息和接口的方法对相应功能进行操作。
