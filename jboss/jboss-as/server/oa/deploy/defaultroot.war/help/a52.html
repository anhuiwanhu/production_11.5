参数设置

在参数设置中，将发文的许多字段都做成了设置项，可以根据用户的实际情况进行灵活的设置。具体如下图：

文件种类——设置文件的类型，如：“制度;规范;决定;规定;办法;通知通报;报告;请示;批复;函;会议纪要;意见”等。文种以半角分号隔开。

秘密级别——文件的保密级别，如：“ ;秘密;机密;绝密”。其中，没有秘密的，又不想显示“无”的，可以用空格。

办理紧急——文件办理的紧急程度，如：“平急;急件;加急;特急;特提”。

内容紧急——内容的紧急程度设置，如：“平急;急件;加急;特急;特提”。

保密期限——文件的保密期限要求，如：“一个月;三个月;一年;短期;长期;永久”。

文号类别——设置系统中发文文号的类型。可以根据单位实际管理情况进行分类，如“总部发文;分公司发文”，“市政府发文;政府部门发文”等。

主题词区域类别——主题词的一级分类。定义格式为：“综合经济|01; 公交、能源、邮电|02; 旅游、城乡建设、环保|03”。类别名称以竖杠“|”和编号以及半角分号隔开。

主题词类别词——主题词的二级分类。定义格式为：“计划|01A; 经济管理|01B; 工业|02A; 交通|02B; 能源|02C; 邮电|02D; 旅游|03A; 服务业|03B; 城乡建设|03C; 环保|03D”。从这个设置可以看出，“计划”是属于之前定义的主题词区域类别中编号为01即“综合经济”下的类别词。该区域类别下还有其他类别词，即设置为：“经济管理|01B”。

公开属性——文件的公开属性，如：“主动公开;依申请公开;不予公开”。

我的收文允许复制——设置“我的收文”中打开的正文内容是否允许复制。未勾选：正文内容不可复制；勾选：正文内容可以复制。

查看正文允许复制——设置点击“查看正文”按钮时打开的正文内容是否允许复制。未勾选：正文内容不可复制；勾选：正文内容可以复制。

【注意】定义内容时，每个选择项之间请以半角分号（即英文输入法下的分号）隔开。

其中，“签发日期”套入模板时，显示中文日期，如“二〇〇九年二月九日”，而不是阿拉伯数字。“印发日期”套入模板时，显示阿拉伯数字，如“2009年2月13日”。

主题词设置

按照国家标准的主题词分类方式进行管理。总体分为主表和附表两大类。

每一类词表分为三个层次。第一层是对主题词区域的分类，如“综合经济”、“财政、金融”类等。第二层是类别词，即对主题词的具体分类，如“综合经济”类中的“计划”、“经济管理”等。第三层是类属词，如“规划 统计 指标”等，设置时以一个空格隔开。第二层和第三层统称为主题词，用于文件的标引。

点击模板设置，如下图所示：

通过点击“新建Word模板”按钮进行模块设置的操作，点击“新建Word模板”按钮后，系统可能会提示安装控件，用户成功安装控件后方可使用。

 

如何定义公文模板？

1、公文首先包含的是文件头，文件头的制作方式有两种：

如文件头格式比较简单，可以直接通过在Word模板中输入内容，同时通过字体、字号、颜色等设置调整为理想状态。

如文件头格式、字体比较特殊，直接在Word进行输入编辑无法实现，可以通过图片编辑软件将文件头制作成图片，在通过Word中自带的图片插入功能添加至模板中。

2、接下来就是“文号”了，将光标放置在适当位置，然后点击页面右侧工具栏中的“定义标签”按钮，如下图所示：

通过点击下拉框箭头选择到“[文号][文号]”标签，再点击添加。之后再看看Word模板，“文号”标签已经插入到光标所在的位置。可以为模板中的所有文字或标签进行字体设置。这样，在生成正式文件的时候，就会将这些标签在发文表单上对应字段的数据提取到正文中，并且设定的字体显示。

如何使用套打功能？

当我们公文正式成文后是包含文件头、印章的保护文档，用户无法对正文内容、文件头，印章进行拷贝粘贴的操作，可是我们单位已经有很多包含文件头的纸张，用于公文的套打操作，可现在正文内容已经保护了，如何进行套打操作呢？

首先在设置机关代字的时候，将红头作为图片设置，然后在模板设置中，插入“红头”标签。这样在正文中，就可以通过“套打”或“显示/隐藏红头”来进行套打操作。

注：文件头和红线必须分开制作，不能做在同一张图片内，因为文件头和图片中间一般都有“文号”。

 

表单设置

通过表单设置，可以自定义发文表单（支持发文新流程ezFlow的表单）。在流程设置的时候可以选择所定义的表单。

其中，表单设置中的备用字段，需通过在用户数据表的发文管理数据表中新增发文表单的备用字段，通过显示设置选择字段属性。备用字段在新增表单页面右侧的“主表字段”中列出。

点击“新增”按钮，显示如下页面。和系统管理→自定义平台→自定义表单的操作方法相同，可以在word中事先排版好发文表单，然后在下图的HTML编辑器中将光标居中，将word中的表单复制过来。

将光标定位到合适的单元格中，从右侧选择对应的字段。其中有几个预留字段，可以选用。全部对应之后，保存退出即可。系统会自动创建一个相同的打印表单，通过点击“显示模版”、“打印模版”，可以分别修改显示或打印模块。

点击“新增”按钮，显示如下页面：

l 可维护人——设置此表单的维护人。拥有发文设置权限的用户，维护权限范围不包含此记录，但只要在可维护人范围内，同样可以修改或删除此表单。

l 同步到打印模板——勾选此选项，保存时会同时将打印模板进行修改。

 

【特别说明】字段“参考文件”，用于上传发文的参考文件或发文依据，和“附件”字段的区别是：只是在待办文件、在办文件等文件办理各栏目中和办理查阅中，可以看到参考文件，但是最终成文分发后在我的收文中不显示参考文件。即，参考文件不随公文分发。

字段“流程名称”，用于在表单最上方是否显示流程名称，比如流程名称为“万户网络技术有限公司发文”，则在发文表单最上方显示“万户网络技术有限公司发文”。

分发单位

分发单位主要用于“分发”操作时选择分发范围。分发公文时，默认分发范围自动取主送、抄送选择的分发单位设置的相关组织或人员。这样就避免分发时再反复选择分发对象了。

文号设置

文号与机关代字是相关联的，选择了机关代字，流程的编号人员在编号这一步，根据机关代字对应的文号规则进行编号。如下图：

l 文号类别字段如已在“参数设置”中设置完毕，此时可以通过下拉框看到选择项了，如出现多个选择项出现在同一行，请检查分隔符输入是否正确。（应为半角状态下的分隔符，即英文输入法状态时输入。）

l 文号名称按照实际需要进行设置。

l 排序码：“文号设置”的列表界面按照设置的排序码进行排序，选择文号的页面也需要按照设置的排序码进行排序。

 

l 机关代字：此处可以暂时不选择，等设置机关代字时再通过机关代字与之关联也可。

l 文号中序号跨年重排：即到下一年度的时候，文号从1开始。

l 文号中序号的位数：默认为1。

l 序号初始值：默认为1。

l 格式中是否有年度：一般都是有年度的，如“阜政发﹝2008﹞1号”。其他定期发布的“政务信息”、“政务要情”、“会议纪要”等，其编号可能为“第1期”，如果也通过发文处理，则“格式中是否有年度”设置为否。

l 格式中是否有机关代字：同上。

l 文号格式：一般默认为“机关代字﹝年度﹞序号号”。其中“机关代字”、“年度”、“序号”为变量，六角号“﹝﹞”和最后的“号”为常量。如果编号为“第*期”，则将“格式中是否有年度”和“格式中是否有机关代字”设置为否，“文号格式”设置为“第序号期”即可。设置后的文号格式可以通过预览按钮进行查看。

l 可维护人——设置此文号的维护人。拥有发文设置权限的用户，维护权限范围不包含此记录，但只要在可维护人范围内，同样可以修改或删除此文号。

注：一个机关代字只能对应一个文号；相反，一个文号可以对应多个机关代字。

流水号设置

流水号与文号功能类似。主要是用于公文归档，如无公文归档需求，可以不输入。

l 流水号名称：按照实际需要进行设置。

l 机关代字：此处可以暂时不选择，等设置机关代字时再通过机关代字与之关联也可。

l 流水号中顺序号的位数：默认为4。

l 文号中序号跨年重排：即到下一年度的时候，序号从1开始。

l 格式中是否有年度：一般都是有年度的，如“﹝2008﹞1号”。

l 顺序号的初始值：默认为1。

l 流水号格式：一般默认为“﹝年度﹞顺序号”。其中 “年度”、“顺序号”为变量。设置后的流水号格式可以通过预览按钮进行查看。

l 可维护人——设置此流水号的维护人。拥有发文设置权限的用户，维护权限范围不包含此记录，但只要在可维护人范围内，同样可以修改或删除此流水号。

 

注：可以对多个机关代字统一编流水号，比如可以对所有以市政府名义发出的文（多个机关代字）编统一序列的流水号。例如，阜政发﹝2008﹞12号（文号）的文件编2008018号（流水号），随后发出的阜政函﹝2008﹞3号的文件编2008019号。

机关代字设置

机关代字与文号、公文模版、流程以及流水号是多对多的关系。在流程流转时选择了机关代字，点击“生成正式文件”时，如果有多个模板则弹出模板的选择页面，显示出此机关代字所对应的模板供用户选择。

l 机关代字——按照实际需要进行设置。

l 排序码——按照需要输入排序码，可对机关代字进行排序设置。

l 文号——选择机关代字相对应的文号；

l 流水号——选择机关代字相对应的流水号；

l 对应模板——选择机关代字相对应的公文模板；

l 对应流程——选择使用该机关代字的发文流程。可以选择多个流程，即一个机关代字可以对应多个流程。

l 使用范围——哪些部门可以使用该机关代字。

l 对应红头——在上面“模板设置”中提到，如果要使用套打功能，则需要将红头作为图片，在此导入。另外，更有普遍意义的应用是：如果一个庞大组织架构的单位，其中很多分子单位使用相同的模板，那么我们可以将红头（制作成图片）作为标签插入模板，这样就只需要设置一个模板即可，而不需要因为红头的不同而为每个单位建立一个模板。

l 可维护人——设置此机关代字的维护人。拥有发文设置权限的用户，维护权限范围不包含此记录，但只要在可维护人范围内，同样可以修改或删除此机关代字。

流程设置

流程设置——公文流程的使用和设置方式可参照《工作流程》一章。此处着重介绍一下与工作流程按钮有别的专属公文操作的按钮。

专属发文的按钮如下图所示：

其中，“待办按钮”指在待办文件中出现的按钮，“通用按钮”指在文件办理中的各个栏目中都可用的按钮。各按钮解释如下：

l 加入督办任务：用户在办理发文的时候，可以同时将此文加入到催办督办中。

l 起草正文：此按钮一般情况只出现在拟稿人发起流程时，用户填写公文正文内容之用，其他步骤无法使用。

l 批阅正文：用于公文流程办理人批阅公文正文之用，对公文正文可以进行修改，修改之后默认记录了修改痕迹，但不显示。如需要查看修改痕迹，可点击Word控件上方的按钮。

l 生成正式文件：此按钮与起草正文、批阅正文功能类似，都对正文拥有写权限，可以进行修改内容的操作。区别是，生成正式文件之前，需要先选择机关代字，因为机关代字对应唯一的模板，所以在点击“生成正式文件”按钮后，在打开的正文中，系统自动套用模板，生成了包含红头、文号、正文、印章等要素的正式文件。

l 编号：用户对公文进行编号操作，一般此按钮定义在公文编号步骤。

l 分发：点击“分发”按钮后，在弹出的页面上选择用户，可以将最终成文发送给这些用户。这些用户可以在“我的收文”中收到此文。

l 转收文：用户在办理发文的时候，可以将此文转为收文流转。

l 转文件送审签：用户在办理发文的时候，可以将此文转为送审签流转。

l 公文交换：可将此发文发送到其他分子公司的系统中，从而实现不同单位之间的公文传递。

l 查看正文：对公文正文内容拥有只读操作。正文内容一般为拟稿人填写，此按钮可出现在“文件办理”中的各种状态内的文件。

l 再次套红：不管当前的文件是否已经被生成正式文件，都会根据已经设置好的红头模板再次生成一份红头文件。（该按钮可以解决‘在发文中，某份文件一旦被生成过正式文件并保存后，再次点击“生成正式文件”时，就套不出红头’的问题。）

5.2.3办理查阅

在办理查阅中，具有“发文管理-办理查阅-查看”权限的用户，可以查看范围内的发文，包括在办文件和办结文件。

具有“发文管理-办理查阅-维护”权限的用户，可以修改、删除、强制结束发文流程，并可以将发文发布到信息管理栏目下。

比如，有的用户把文件批下去了，才想起来写了一个错字或者忘记填写主题词，这时候，可以要求有权限的公文管理员在办理查阅中修改它。

又如，有的用户在办理发文流程中，确认无需再走后续步骤，管理员可以在办理查阅中强制结束发文流程，发文自动办理完毕。
